# Pulse

Pulse — клиент-серверное веб-приложение интернет-магазина спортивного инвентаря.  
Проект реализует базовый сценарий e-commerce: просмотр каталога, поиск и фильтрацию товаров, работу с корзиной, оформление заказов, управление профилем пользователя и административную панель для управления данными.

## Функциональность

### Для пользователя
- регистрация, вход и выход из аккаунта;
- просмотр каталога товаров на главной странице;
- поиск товаров по названию;
- фильтрация товаров по категории и цене;
- просмотр карточки товара;
- добавление товаров в корзину;
- изменение количества товаров в корзине и удаление позиций;
- оформление заказа;
- просмотр истории заказов и деталей заказа;
- отмена заказа;
- заполнение личного профиля: ФИО, возраст, номер карты, адрес доставки;
- добавление собственного товара на платформу.

### Для администратора
- просмотр пользователей и их статусов;
- управление пользователями;
- добавление и редактирование категорий и товаров;
- просмотр заказов и информации о заказанных товарах;
- работа через встроенную Django admin panel.

## Технологический стек

- **Backend:** Python 3.10, Django 5.2.1
- **Frontend:** HTML, CSS, Bootstrap
- **Database:** SQLite
- **Authentication:** сессионная аутентификация Django
- **Tools:** Git, venv

## Структура базы данных

В проекте реализована реляционная база данных со связями между основными сущностями:

- `User`
- `UserProfile`
- `Cart`
- `CartItem`
- `Order`
- `OrderItem`
- `Category`
- `Item`
- `Discount`
- `Review`

База данных покрывает основные бизнес-процессы интернет-магазина: хранение пользователей, товаров, категорий, корзин, заказов и отзывов.

## Основные маршруты

### Аутентификация и профиль
- `GET, POST /signup/` — регистрация пользователя
- `GET, POST /login/` — вход в систему
- `GET /logout/` — выход из системы
- `GET /myaccount/` — просмотр профиля
- `POST /myaccount/` — обновление профиля

### Товары
- `GET /items/browse/` — поиск и фильтрация товаров
- `GET /items/<int:pk>/` — страница товара
- `GET, POST /items/new/` — добавление нового товара

### Корзина
- `POST /items/add-to-cart/<int:pk>/` — добавить товар в корзину
- `GET /items/cart/` — просмотр корзины
- `POST /items/remove-from-cart/<int:item_id>/` — удалить товар из корзины
- `POST /items/update-cart-item/<int:item_id>/` — обновить количество товара

### Заказы
- `GET, POST /checkout/` — оформление заказа
- `GET /orders/` — история заказов
- `GET /orders/<int:pk>/` — детали заказа
- `POST /orders/<int:pk>/cancel/` — отмена заказа

### Дополнительные страницы
- `/about/`
- `/contact/`
- `/privacy/`
- `/terms/`

## Что реализовано в проекте

- полноценный пользовательский сценарий от регистрации до оформления заказа;
- фильтрация и поиск по каталогу;
- личный кабинет с сохранением данных для доставки и оплаты;
- возможность размещения нового товара пользователем;
- административная панель для управления сущностями;
- модульная структура, которую можно расширять дальше.
